# Universal Brain

> "An AI Operating System that grows, learns, remembers, and evolves with its user."

---

# Vision

Universal Brain is **not another AI assistant.**

It is **not another chatbot.**

It is **not another coding agent.**

Universal Brain is a lifelong, local-first AI Operating System whose intelligence is independent of any specific AI model.

The purpose of Universal Brain is to become a permanent digital brain that both the user and AI continuously build together.

The system should become more knowledgeable every day without becoming dependent on Claude, GPT, Gemini, or any future model.

The user owns the knowledge.

The AI assists in organizing, understanding, expanding, and connecting that knowledge.

---

# Core Philosophy

Models are temporary.

Knowledge is permanent.

Memory belongs to the user.

Markdown is forever.

The AI is replaceable.

The Brain is not.

---

# Mission

Build an AI system that:

- Learns continuously.
- Organizes knowledge automatically.
- Connects ideas.
- Improves itself over time.
- Helps users think better rather than think for them.
- Works with any language model.
- Can survive technological changes for many years.

---

# What Universal Brain Is

Universal Brain is an AI Operating System composed of independent modules.

These modules include:

- Planning
- Memory
- Knowledge Management
- Context Building
- Retrieval
- Reflection
- Learning
- Tool Usage
- Model Routing
- Skill Execution

The intelligence belongs to these modules.

The language model only performs reasoning.

---

# What Universal Brain Is NOT

Universal Brain is NOT:

- Claude Code
- Cursor
- Codex
- Antigravity
- Copilot
- OpenHands
- A chatbot
- A note-taking application
- A wrapper around an LLM

Those are interfaces or reasoning engines.

Universal Brain owns the architecture.

---

# Long-Term Memory

Obsidian is the permanent brain.

Every Markdown note represents a knowledge node.

Every wiki link represents a relationship.

The graph formed by these notes is the AI's long-term knowledge graph.

The vault is the single source of truth.

The AI must never invent permanent knowledge without explicitly storing it.

---

# Memory Layers

Universal Brain contains multiple memory systems.

## Working Memory

Current task.

Short-lived.

---

## Conversation Memory

Current interaction.

---

## Project Memory

Knowledge related to the active project.

---

## Long-Term Memory

Stored inside Obsidian.

---

## Semantic Memory

Embeddings.

Used for retrieval.

---

## Procedural Memory

How to perform tasks.

---

## Reflection Memory

Lessons learned from previous work.

---

# Knowledge

Knowledge should always be:

- Human readable
- Markdown based
- Searchable
- Version controlled
- Interconnected
- Explainable

Knowledge should never become trapped inside vector databases or proprietary formats.

Embeddings exist only to improve retrieval.

Markdown remains the truth.

---

# Design Principles

## Local First

The user's knowledge always belongs to the user.

---

## Model Agnostic

No feature should depend on one vendor.

Replacing Claude with GPT or Gemini should require changing only one adapter.

---

## Modular

Every subsystem should be independently replaceable.

---

## Transparent

The AI should explain why it makes decisions.

No hidden reasoning pipelines.

---

## Extensible

New models.

New tools.

New skills.

New interfaces.

Should all plug into the existing architecture.

---

## Durable

Universal Brain should remain useful for years.

Avoid trendy architectures.

Favor stable technologies.

---

# The Planner

The planner is the executive brain.

Nothing should directly call an LLM.

Every request flows through the planner.

The planner decides:

- What information is needed.
- Which memory to access.
- Which skills to load.
- Which tools to execute.
- Which model to use.
- Whether reflection is necessary.

---

# Skills

Skills are independent capabilities.

Examples:

- Linux
- Python
- Docker
- Git
- Networking
- Cybersecurity
- Research
- Browser
- Terminal
- Obsidian
- Memory

Skills should never know which LLM is running.

---

# Models

Models are interchangeable reasoning engines.

Supported examples:

- Claude
- GPT
- Codex
- Antigravity
- Gemini
- Ollama
- Future local models

Universal Brain should never depend on any model-specific behavior.

---

# Retrieval

Universal Brain should never load the entire knowledge base.

Instead:

Search.

Retrieve.

Expand relationships.

Rank relevance.

Build context.

Reason.

---

# Self Improvement

After every completed task Universal Brain should ask:

What was learned?

Can documentation improve?

Can relationships improve?

Should knowledge be reorganized?

Can summaries improve?

Can duplicate notes merge?

Learning is continuous.

---

# User Experience

The user should feel like they are interacting with one evolving intelligence.

Not switching between multiple AI products.

The interfaces may change.

The brain remains the same.

---

# Success Criteria

Universal Brain succeeds if:

- Knowledge continuously improves.
- Switching language models requires minimal effort.
- The Obsidian graph becomes richer over time.
- The AI becomes more helpful as the knowledge base grows.
- Every project benefits from previous experience.
- The user owns every piece of long-term knowledge.

---

# Engineering Principles

- Simplicity before complexity.
- Composition over inheritance.
- Interfaces over implementations.
- Testability over cleverness.
- Explicit over implicit.
- Documentation is part of the product.
- Every module should have a single responsibility.

---

# Guiding Question

Before implementing any feature, ask:

"Does this make the Brain smarter, more durable, and more independent?"

If the answer is no,

don't build it.

---

> Build the Brain once.

> Replace the Models forever.
